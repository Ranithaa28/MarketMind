# Contributing to MarketMind

First off, thank you for considering contributing to MarketMind! It's people like you that make this tool such a great resource for founders and entrepreneurs.

## Code of Conduct

By participating in this project, you agree to abide by our Code of Conduct. We expect all contributors to maintain a welcoming and inclusive environment.

## Getting Started

1. **Fork the repository** on GitHub.
2. **Clone your fork** locally: `git clone https://github.com/your-username/MarketMind.git`
3. **Create a new branch** for your feature or bug fix: `git checkout -b my-new-feature`
4. **Follow the setup instructions** in the `README.md` to run the project locally using Docker or manually.

## Development Workflow

- **Backend**: We use FastAPI and LangGraph. Ensure your code passes all `pytest` checks.
- **Frontend**: We use Next.js App Router and Tailwind CSS. Ensure your UI matches the design system and passes `npm run build`.
- **Commits**: Please use descriptive commit messages. We recommend the [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) specification.

## Pull Requests

1. Ensure your code is fully tested.
2. Update the `README.md` if you introduce any new environment variables or architectural changes.
3. Open a Pull Request against the `main` branch.
4. Provide a clear description of the problem you're solving and the changes you've made.

## Bug Reports and Feature Requests

If you find a bug or have a feature idea, please open an issue in the GitHub repository using the provided issue templates.

Thank you for contributing! 🚀
